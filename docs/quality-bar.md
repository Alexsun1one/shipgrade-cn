# Quality Bar

## Done Means Evidence

`done` 必须同时满足:

1. 结果存在: 文件、服务、模型、配置或文档可被打开。
2. 验证存在: 命令、测试、截图、日志、探针或人工检查点。
3. 边界存在: 说明哪些没有覆盖。
4. 接手存在: 下一位 agent 能从明确路径继续。

## Ship-Grade Checklist

- [ ] 用户目标被复述成可验收结果。
- [ ] 当前事实和旧记忆被区分。
- [ ] 源代码/配置改动最小且可解释。
- [ ] 许可证和来源边界保留。
- [ ] secrets/session/auth/cookie 不进入产物。
- [ ] 至少一个相关验证已运行。
- [ ] 失败或跳过的验证写清原因。
- [ ] 文档/记忆/handoff 已更新。

## Red Flags

- 只给计划,没有产物。
- 只给产物,没有验证。
- 只给验证,没有用户价值。
- 只给指标,没有探针或样例。
- 只说“世界级”,没有质量门槛。
